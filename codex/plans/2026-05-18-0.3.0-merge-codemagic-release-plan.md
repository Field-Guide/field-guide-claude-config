# 0.3.0 Merge And Codemagic Release Plan

## Goal

Merge the current release branch, verify the updated GitHub CI path, bump Field
Guide to `0.3.0`, and trigger the tag-driven Codemagic beta workflows for iOS
TestFlight and Android Firebase App Distribution.

## Checklist

- [ ] Confirm the working tree is clean and record current branch topology.
- [ ] Push any local branch state required for GitHub checks.
- [ ] Verify GitHub CI reports the updated quality gate accurately, including
  the fixed analyzer step and removed soak/local backend paths.
- [ ] Merge `fix/entry-range-export-form-pdf-photo-naming` into `main` without
  discarding the existing local `main` commit.
- [ ] Update app version from `0.2.9+6` to `0.3.0+1`.
- [ ] Run local release checks before tagging.
- [ ] Commit the `0.3.0+1` version bump with release evidence.
- [ ] Push `main`.
- [ ] Create and push tag `field-guide-beta-v0.3.0+1`.
- [ ] Confirm Codemagic starts `ios-testflight` and `android-firebase`.
- [ ] Monitor Codemagic build status and record build URLs/results.

## Notes

- Codemagic workflows are tag-driven through `field-guide-beta-v<version>+<build>`.
- `pubspec.yaml` and the release tag must match exactly.
- The maintained backend verification path is live backend/device evidence;
  local Docker/local Supabase is intentionally not part of this release gate.
