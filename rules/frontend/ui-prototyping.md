# UI Prototyping

- For new screens or major redesigns, offer a quick HTML mockup before writing Flutter code.
- When available, use the HTML sync and browser tooling to iterate visually with the user.
- Keep mockups phone-sized unless the target surface is explicitly tablet or desktop.
- Use the app palette and realistic construction-domain content, not generic placeholder copy.
- Include the same major navigation chrome the production screen will have.
